# GitHub Repository Management Skill

## Purpose
Create, manage, and publish code repositories on GitHub using CLI and browser automation.

## Prerequisites
- `git` installed and configured
- `gh` (GitHub CLI) authenticated: `gh auth login`
- SSH key or token configured for push access

## Workflows

### Create New Repository and Push Code
```
1. Create local project:
   mkdir my-project && cd my-project
   git init

2. Add code files (generated by coding-skill)

3. Create README.md with project description

4. Initialize git:
   git add -A
   git commit -m "Initial commit: project description"

5. Create GitHub repo:
   gh repo create my-project --public --source=. --push

6. Verify: gh repo view --web
```

### Push Updates
```
1. Stage changes:
   git add -A

2. Commit with descriptive message:
   git commit -m "feat: description of changes"

3. Push:
   git push origin main
```

### Create a Release
```
1. Tag the release:
   git tag -a v1.0.0 -m "Release v1.0.0: description"

2. Push tag:
   git push origin v1.0.0

3. Create GitHub release:
   gh release create v1.0.0 --title "v1.0.0" --notes "Release notes here"
```

### Fork and Contribute
```
1. Fork repo:
   gh repo fork owner/repo --clone

2. Create branch:
   git checkout -b feature/my-feature

3. Make changes and commit

4. Push:
   git push origin feature/my-feature

5. Create PR:
   gh pr create --title "Feature: description" --body "Details..."
```

### Manage Issues
```
1. List issues:
   gh issue list

2. Create issue:
   gh issue create --title "Bug: description" --body "Steps to reproduce..."

3. Close issue:
   gh issue close ISSUE_NUMBER
```

## Commit Message Convention

Use conventional commits:
- `feat:` - New feature
- `fix:` - Bug fix
- `docs:` - Documentation changes
- `style:` - Code style changes (formatting, etc.)
- `refactor:` - Code refactoring
- `test:` - Adding or updating tests
- `chore:` - Maintenance tasks

## JavaScript Snippets (via gh CLI)

### Create Repo and Push
```bash
#!/bin/bash
PROJECT_NAME=$1
DESCRIPTION=$2

mkdir -p "$PROJECT_NAME"
cd "$PROJECT_NAME"
git init
echo "# $PROJECT_NAME" > README.md
echo "$DESCRIPTION" >> README.md
git add -A
git commit -m "feat: initial project setup - $DESCRIPTION"
gh repo create "$PROJECT_NAME" --public --source=. --push --description "$DESCRIPTION"
echo "Repository created: https://github.com/$(gh api user -q .login)/$PROJECT_NAME"
```

### Quick Project Setup
```bash
#!/bin/bash
# Initialize a Node.js project with common defaults
npm init -y
echo "node_modules/" > .gitignore
echo "dist/" >> .gitignore
echo ".env" >> .gitignore
git init
git add -A
git commit -m "chore: initial project setup"
```

## Troubleshooting

- **Auth issues**: Run `gh auth status` to check authentication
- **Push denied**: Verify SSH key or token permissions
- **Repo exists**: Use `gh repo clone` instead of create
- **Rate limits**: GitHub API has rate limits; check with `gh api rate_limit`
